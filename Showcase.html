Tic-Tac-Toe Game Written in C++

As a Computer Science major, I am often asked to showcase my programming abilities. This was done frequently with a C++ programming course I took last semester. In this course, I learned about the basic programming concepts that every programmer should know. By this point, other CS students and I should be familiar a lot of these concepts. This class is different because I was learning a new language.

There were plenty of programming assignments in this class, but the one I think was the funnest was creating a tic-tac-toe game. For my midterm, I had to create a tic-tac-toe game that swaps back and forth between two players and asks the players if they want to continue playing after a match has ended.



Playing the Game

When the game starts, player 1 is provided with a 'blank' tic-tac-toe game. The nine possible slots are numbered 1-9, and if a player wants to make a move, they have to select that number.



Player 1 is asked to make a move. They make a move based on the 'blank' tic-tac-toe board. By selecting a number, player 1 fills that slot with an 'X', and then it's player 2's turn (who uses 'O's). The game goes on until someone wins, or no one wins.



When a player wins, a congratulations message appears and asks the players if they would like to play another round (player 2 didn't do too hot this round).



When no one wins, a message appears saying that no one has won, and asks the players if they would like to play another round. This sequence goes on endlessly! Until they decide that they've had enough.



Writing the Code

It seems like a simple game, right? The rules are simple, and the execution is clean... So how much code could such an easy game take? The answer is a lot!

By the time I was finished writing my code, I used just under 300 lines. It takes a lot of thinking to get such a simple thing done! It's part of the puzzle, and that's why I enjoy it! Below is a link to my code. I've included it as a text file for simplicity. Thanks for viewing with me!
